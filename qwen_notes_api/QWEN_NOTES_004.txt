### **Confirmed Components:**

#### **1. Engine Modules**
- **Ollama (`ollama`):** Designed for Apple Silicon and certain NVIDIA GPUs.
  - **Best For:** Macs (M1/M2/M3/M4) running MLX-format or GGUF models natively.
  - **API Endpoint:** `/v1/chat/completions`
  
- **VLLM (`vllm`):** Suitable for large-scale datacenter NVIDIA GPUs like A100, H100, H200, L40, A10, and A30.
  - **Best For:** Running models too large for a single device by distributing across multiple heterogeneous machines.
  - **API Endpoint:** `/v1/chat/completions`
  
- **LlamaCpp (`llamacpp`):** Default engine when no GPU is detected.
  - **Best For:** General-purpose inference on CPUs and systems without dedicated GPUs.
  - **API Endpoint:** `/v1/chat/completions`
  
- **MLX (`mlx`):** Optimized for Apple Silicon Macs.
  - **API Endpoint:** `/v1/chat/completions`

- **LM Studio (`lmstudio`):** Connects to the LM Studio desktop application’s built-in server, which exposes an OpenAI-compatible API.
  - **Best For:** Users who prefer a GUI for model management and want a zero-configuration local server.
  - **API Endpoint:** `/v1/chat/completions`
  
- **Exo (`exo`):** Distributed inference runtime that partitions model layers across multiple heterogeneous devices.
  - **Install:** `pip install exo`
  - **Best For:** Running models too large for a single device by distributing across multiple Apple Silicon or heterogeneous machines.
  - **API Endpoint:** `/v1/chat/completions`

- **Nexa (`nexa`):** On-device inference server via a FastAPI shim that wraps `nexaai.LLM`.
  - **Install:** `pip install nexaai`
  - **Best For:** On-device inference with GGUF models on Apple Silicon or CPU.
  - **API Endpoint:** `/v1/chat/completions`

- **Lemonade (`lemonade`):** Optimized for AMD consumer GPUs and Ryzen AI Neural Processing Units (NPUs).
  - **Install:** Visit [lemonade-server.ai](https://lemonade-server.ai/) for platform-specific installation instructions.
  - **Best For:** Ryzen AI GPUs and NPUs, and AMD-based desktop and laptop systems.
  - **API Endpoint:** `/v1/chat/completions`

- **Uzu (`uzu`):** Connects to the Uzu inference runtime.

  - **Install:** `pip install python-apple-fm-sdk`
  - **Best For:** Running Apple Foundation Models natively on Apple Silicon hardware.
  - **API Endpoint:** `/v1/chat/completions`

#### **2. Engine Discovery and Configuration**
- **`_discovery.py`:** Contains functions for finding and instantiating engines at runtime:
  - `get_engine(config, engine_key=None):` Returns a `(key, engine_instance)` tuple for the requested engine or falls back to available healthy engines.
  - `discover_engines(config):` Probes all registered engines for health and returns a sorted list of healthy `(key, engine)` pairs.
  - `discover_models(engines):` Calls `list_models()` on each engine and returns a dictionary mapping engine keys to model ID lists.

- **Configuration File (`~/.openjarvis/config.toml`):** 
  - Engine hosts and defaults are configured using nested per-engine sub-sections.
  - Example:
    ```toml
    [engine]
    default = "ollama"
    ```

### **2. Hardware Auto-Detection**
- **Detection Methods:**
  - **NVIDIA GPU:** Uses `nvidia-smi` to extract GPU name, VRAM (GB), and count.
  - **AMD GPU:** Utilizes `rocm-smi` to get the GPU name.
  - **Apple Silicon:** Employs `system_profiler SPDisplaysDataType` to retrieve the chipset model name.
  - **CPU:** Leverages `/proc/cpuinfo` or `sysctl` to obtain the brand string.
  - **RAM:** Retrieves total GB using `/proc/meminfo` or `sysctl hw.memsize`.

### **3. Engine Compatibility**
- **OpenAI Compatibility Layer (`_OpenAICompatibleEngine`):** Provides a shared implementation for engines that serve the standard `/v1/chat/completions` endpoint.
  - **Key Behaviors:**
    - **Synchronous Generation:** Sends requests to `POST /v1/chat/completions` with `stream=False`.
    - **Streaming:** Uses `POST /v1/chat/completions` with `stream=True`, parsing SSE `data:` lines.
    - **Model Listing:** Retrieves model IDs from `GET /v1/models`.
    - **Health Check:** Probes `GET /v1/models` with a 2-second timeout.
    - **Tool Call Fallback:** Retries requests without tools if the engine does not support function calling.
