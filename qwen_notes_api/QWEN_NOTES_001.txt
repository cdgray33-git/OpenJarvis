3. **Detailed Agent Descriptions:**

   - **OrchestratorAgent:**
     - Multi-turn agent using a tool-calling loop.
     - Uses OpenAI's function-calling format for tool invocation.
     - Loops until a final answer is produced or the maximum number of turns is exceeded.

   - **NativeReActAgent:**
     - Implements a Thought-Action-Observation (ReAct) loop.
     - Parses structured text outputs from the LLM to drive tool execution.
     - Case-insensitive parsing for better compatibility.

   - **NativeOpenHandsAgent:**
     - CodeAct-style agent that generates and executes Python code.
     - Handles context window overflow with automatic truncation.
     - Can also invoke tools via `Action:` / `Action Input:` output.

   - **RLMAgent:**
     - Recursive Language Model agent based on RLM paper.
     - Stores context as a variable in a persistent REPL.
     - Uses recursive sub-LM calls for processing context.

   - **OpenHandsAgent (SDK):**
     - Wrapper around the `openhands-sdk` package.
     - Requires Python 3.12+ and is an optional dependency.

4. **ToolExecutor:**
   - Handles tool execution, particularly in agents like `OrchestratorAgent`, `NativeReActAgent`, and `RLMAgent`.
   - Converts tools to the appropriate format (e.g., OpenAI function-calling format).

5. **Shared Components:**

6. **System Prompts:**
   - System prompts are built differently for each agent type, ensuring that the LLM understands how to interact with tools and execute actions appropriately.
